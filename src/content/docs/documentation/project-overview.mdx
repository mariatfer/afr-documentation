## What It Does

This project is a production‑grade SSR web platform built with Nuxt 4 and Vue 3.  
It consumes content from a headless CMS, renders pages server‑side for optimal SEO, and includes interactive UI components, dynamic content sections, and form submission with file upload support.

Key capabilities include:

- **Product or content catalog** with category browsing and detail pages
- **Dynamic pages** powered by a headless CMS
- **Contact and request forms** with client‑side validation and file uploads
- **Full SEO instrumentation** (structured data, dynamic sitemap, canonical URLs)
- **Cookie consent system** with event‑driven third‑party activation
- **Interactive UI** (carousels, accordions, scroll‑triggered animations)

| **Fonts** | Self‑hosted webfonts | Zero CLS and consistent rendering |
| **Forms** | Custom validator + FormData | Client‑side validation and submission |
| **Email** | Generic email provider | Form‑to‑email delivery |
| **State** | Pinia + `useState` | Global and per‑page state |
| **Deployment** | Vercel (example) | SSR hosting, analytics, speed insights |

## Key Features

### 1. Dynamic SEO System

Each page defines its own SEO metadata, with global fallbacks.  
Structured data, canonical URLs, and a dynamic sitemap endpoint ensure strong search visibility.

### 2. Generic Service Layer

All API calls go through a typed service registry.  
Adding a new endpoint requires only a service function and a registry entry.

### 3. Cookie Consent Event Bus

Consent is stored in a versioned cookie and broadcast via `CustomEvent`.  
Third‑party components listen for consent events before loading external resources.

### 4. Singleton Card Hover Effect

A single global `mousemove` listener powers hover effects across multiple cards, using `requestAnimationFrame` throttling and reference counting.

### 5. GSAP Scroll‑Reveal Component

A reusable `<ScrollReveal>` component animates content when it enters the viewport, with configurable direction, distance, and scrub.

### 6. Polymorphic Button Component

A single button component renders as a link, external anchor, or native button depending on props — no duplicated markup.

### 7. Type‑Safe Locale Loader

A dynamic JSON loader with TypeScript generics ensures compile‑time safety for translation keys.

### 8. ISR‑Ready Architecture

The project is structured to support incremental static regeneration and SWR‑style revalidation when needed.
